## ⌨️ Горячие клавиши

### Общие
| Клавиша | Действие |
|---------|----------|
| `Ctrl + k` | Выход из режима вставки |
| `Esc` | Снять подсветку поиска |
| `<leader> + s` | Сохранить файл |
| `<leader> + q` | Закрыть окно |
| `<leader> + e` | Открыть/закрыть файловый менеджер (NvimTree) |
| `<leader> + p` | Поиск файлов (Telescope) |

> 💡 `<leader>` = пробел

### Управление табами (Alt + ...)
| Клавиша | Действие |
|---------|----------|
| `Alt + t` | Создать новый таб |
| `Alt + q` | Переключиться на предыдущий таб |
| `Alt + e` | Переключиться на следующий таб |
| `Alt + z` | Закрыть текущий таб |

### LSP (для C/C++ файлов)
| Клавиша | Действие |
|---------|----------|
| `gd` | Перейти к определению |
| `K` | Показать документацию |
| `<leader> + rn` | Переименовать символ |
| `gr` | Найти все использования |

### Отладка (DAP)
| Клавиша | Действие |
|---------|----------|
| `F5` | Запустить/продолжить отладку |
| `F9` | Поставить/убрать точку останова |
| `F10` | Шаг с обходом (step over) |
| `F11` | Шаг с заходом (step into) |
| `<leader> + du` | Открыть/закрыть интерфейс отладчика |

### Автодополнение (nvim-cmp)
| Клавиша | Действие |
|---------|----------|
| `Ctrl + Space` | Ручной вызов автодополнения |
| `Tab` | Выбрать следующий вариант |
| `Shift + Tab` | Выбрать предыдущий вариант |
| `Enter` | Подтвердить выбор |

### DebugPrint
| Клавиша | Действие |
|---------|----------|
| `g?p` | Вставить отладочный print |
| `g?v` | Вставить print с переменной под курсором |

## 📁 LSP (clangd)

### Особенности
- Автоматический запуск для `.c` и `.cpp` файлов
- Фоновая индексация (`--background-index`)
- Поддержка `clang-tidy`
- Поиск корневых маркеров: `compile_commands.json`, `.git`

### Возможности
| Функция | Клавиша |
|---------|---------|
| Переход к определению | `gd` |
| Документация под курсором | `K` |
| Переименование символа | `<leader>rn` |
| Поиск всех использований | `gr` |

## 🐛 Отладка (DAP)

### Настройка
| Параметр | Значение |
|----------|----------|
| Адаптер | `gdb` |
| Формат вывода | `set print pretty on` |

### Конфигурация для C
```lua
name = "Launch"
type = "gdb"
request = "launch"
program = запрашивается при запуске
cwd = "${workspaceFolder}"
```

### Интерфейс отладчика
- Открытие: `<leader> + du`
- Показывает переменные, точки останова, стек вызовов

## 📦 Требования

| Программа | Установка (Arch Linux) |
|-----------|------------------------|
| Neovim >= 0.9.0 | `sudo pacman -S neovim` |
| clangd | `sudo pacman -S clang` |
| gdb | `sudo pacman -S gdb` |
| gcc / g++ | `sudo pacman -S gcc` |
| git | `sudo pacman -S git` |

---

## 🔄 Установка и первый запуск

1. Убедись, что все требования установлены
2. Скопируй конфигурацию в `~/.config/nvim/`
3. Запусти Neovim:
   ```bash
   nvim
   ```
4. Плагины установятся автоматически через `lazy.nvim`
5. При первом открытии C/C++ файла clangd может запросить дополнительную установку

## 💡 Полезные особенности

- **Прозрачность**: фон Neovim прозрачный, виден фон Kitty с размытием
- **Управление табами**: удобные Alt-комбинации (`Alt+t/q/e/z`)
- **Quickfix**: ошибки компиляции отображаются в удобном окне (если настроено)
- **Отладка из коробки**: интеграция с gdb через DAP
